# Git Worktree練習ガイド

## 1. 参考リソース
- [Claude Code時代の新常識！git worktreeで実現する、ストレスフリーな並列開発ワークフロー](https://qiita.com/yonaka15/items/9b0260de6dabaadf13d8) — Git Worktreeの概要から、実務で並列開発に活かすための手法までを日本語で整理したQiita記事。
- [AI開発の並列作業を快適にする git worktree + stgitの連携テクニック](https://zenn.dev/mitzh/articles/3edae8c5235a5e) — Zennで公開されている高度な活用例。Worktreeの応用に触れつつ、周辺ツールとの組み合わせ方を学べます。
- [複数ブランチの同時並行作業にはGit Worktreeを - astamuse Lab](https://lab.astamuse.co.jp/entry/git-worktree) — Worktree運用のメリット整理や実践的なコマンド例がまとまったラボ記事。

## 2. 学習ゴールと前提
- Gitのブランチ運用を理解し、基本操作（clone, branch, checkout, merge）が行える。
- Worktreeを使って「並列開発」「レビュー準備」「検証用ブランチ」の効率化を図ることを目標とする。

## 3. 学習ロードマップ
1. **概念理解** — Worktreeの目的・仕組み・既存ブランチ運用との違いを把握する。参考: Qiita記事の前半、astamuse Lab記事。
2. **基本コマンド習得** — `git worktree add/list/remove/prune` を実際に手を動かして理解する。作業用のダミーリポジトリで試す。
3. **典型ユースケースの体験** — レビュー用ブランチ作成、ホットフィックス対応、CI検証など、代表的なシナリオをWorktreeで再現する。
4. **応用＆ベストプラクティス** — Zenn記事を参考に、他ツール連携やリモート環境での運用、クリーンアップ時の注意点を整理する。
5. **振り返りとドキュメント化** — 学んだポイント・コマンド例・トラブルシュートを本ガイドに追記し、次回以降に参照できる形に整える。
